# Ручне втручання

Нижче зібрано пункти, які не можна коректно вигадувати або завершити без участі автора роботи / кафедри.

## Обов'язкові реквізити

- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: уточнити УДК].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: уточнити точну офіційну назву типу документа на титулі].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: вказати ПІБ студента].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: вказати номер групи].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: вказати ПІБ наукового керівника та, за потреби, консультантів].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: уточнити кафедральні реквізити, протоколи, затвердження, службові дати].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: проставити підписи, дати захисту, дати видачі завдання та інші змінні поля].

## Експерименти та результати

- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: додати реальні результати педагогічного експерименту, якщо він буде проведений].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: додати статистичний аналіз лише після отримання реальних даних].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: підтвердити обсяг функціональної валідації прототипу після фактичного тестування].

## Ілюстративні матеріали

- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: перевірити, чи потрібні офіційно оформлені службові сторінки "Завдання" і "Календарний план" саме для магістерської курсової роботи].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: додати фінальні скріншоти інтерфейсу, якщо вони мають бути отримані після ручного запуску прототипу].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: за потреби замінити плейсхолдери діаграм на фінальні рисунки або вставити згенеровані схеми вручну].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: оновити поле змісту в Word після відкриття документа].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: після фінальних правок повторно порахувати сторінки, рисунки, таблиці, додатки та кількість джерел для анотації].

## Практична частина

- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: встановити `ollama`, завантажити `llama3.1:8b` і виконати ручний E2E-прогін сценарію генерації вправи та фідбеку].
- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: за потреби зафіксувати скріншоти роботи прототипу для розділу 4 або додатків].

## Бібліографія

- [ПОТРІБНО РУЧНЕ ВТРУЧАННЯ: остаточно звірити бібліографічні поля всіх джерел перед поданням, якщо частина записів лишиться не верифікованою автоматично].
