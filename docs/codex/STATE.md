## Що зафіксовано в коді

- Локальний LLM-клієнт оновлено для реального браузерного прогону з `llama3.1:8b`:
  - тайм-аут генерації для Ollama збільшено до 30 с;
  - статус-перевірка винесена в окремий короткий тайм-аут.
- `SessionMemory.resetForExercise()` тепер скидає `firstFailure`, тому JSON-експорт не тягне помилку з попередньої вправи.
- У UI картки вправи показується очікувана назва функції, щоб браузерний сценарій не ламався на прихованому `functionName`.
- Підтримка explain-flow через явний `interactionMode: "explain"` збережена.

## Що зафіксовано в тестах

- `npm test` проходить повністю: 27/27 тестів.
- `npm run check` проходить.
- Додано unit-тест на скидання `firstFailure` між вправами.

## Що зафіксовано в браузерній валідації

- Локальна `Ollama` реально доступна: модель `llama3.1:8b` присутня й відповідає через `http://localhost:11434`.
- Через Playwright пройдено повний сценарій:
  - генерація вправи з джерелом `ollama`;
  - перша невдала спроба;
  - повторна помилка з ескалацією до `targeted_hint`;
  - натискання кнопки пояснення з нейтральним текстом `Що робити далі?`, що переводить систему в `concept_explanation`;
  - експорт JSON-сесії з фактичними ключовими полями.
- Артефакти збережено в `output/playwright/validation/`.

## Залишкові обмеження

- Педагогічний експеримент зі справжніми студентами в межах цієї курсової не проводився.
- Браузерна валідація підтверджує технічну працездатність, але не дає підстав для висновків про педагогічну ефективність.
